# 2026-06-09 Dynamic Weather Tip Design Spec

## Overview
This specification details the transition of the "Daily Weather Tips" ("每日气象贴士") card on the homepage from a static, date-modulo frontend mock array to a dynamic, LLM-generated, and weather-aware system.

## Goals
1. Make the daily weather tip relevant to the actual current weather conditions of the queried city.
2. Maintain backward compatibility and fallback mechanisms: if the LLM generation fails or the user has not drawn a city yet, fallback to the calendric 12-tip cycle.
3. Ensure types and schemas are fully synchronized between frontend and backend.

## Architectural Changes

### 1. Backend Data Schema

File: `backend/app/schemas/assistant.py`

Add a new schema for the weather tip and include it in `WeatherCardResponse`:
```python
class WeatherOracleTip(BaseModel):
    title: str
    advice: str

class WeatherCardResponse(BaseModel):
    # Existing fields
    city: str
    date: str
    timezone: str
    updated_at: str
    weather: WeatherOracleWeather
    tarot: WeatherOracleTarot
    fortune: WeatherOracleFortune
    mood_guide: WeatherOracleMoodGuide
    daily_advice: WeatherOracleDailyAdvice
    weather_mappings: List[WeatherOracleMapping]
    
    # New Field
    weather_tip: Optional[WeatherOracleTip] = None
```

### 2. Backend Service & Endpoint Logic

File: `backend/app/routers/assistant.py`

- **Static Helper for Fallback Weather Tips**:
  Define the 12 weather tips in the backend so they can be selected dynamically by day of the month as a fallback.
- **LLM Prompt Modification**:
  Update `build_weather_oracle_prompt` to instruct the LLM to generate `weather_tip` containing `{title, advice}` matching the weather condition of the city.
- **Data Normalization**:
  Update `normalize_weather_oracle_model_data` to extract and validate `weather_tip` from the LLM JSON response. If missing, populate it with the computed fallback weather tip.

### 3. Frontend Types

File: `src/types/weatherOracle.ts`

- Add `WeatherOracleTip` interface:
  ```typescript
  export interface WeatherOracleTip {
    title: string
    advice: string
  }
  ```
- Update `WeatherOracleReading` interface:
  ```typescript
  export interface WeatherOracleReading {
    // ...
    weather_tip?: WeatherOracleTip
  }
  ```

### 4. Frontend Components & Views

- **Parent View (`src/views/WeatherOracle.vue`)**:
  Pass the dynamic `weather_tip` from the active reading to the left sidebar component:
  ```vue
  <OracleLeftSidebar :weather-tip="reading?.weather_tip" />
  ```

- **Sidebar Component (`src/components/oracle/OracleLeftSidebar.vue`)**:
  - Accept `weatherTip` as a prop:
    ```typescript
    const props = defineProps<{
      weatherTip?: WeatherOracleTip
    }>()
    ```
  - Define a computed property `displayTip` that returns `props.weatherTip` if present and valid; otherwise, it falls back to the day-of-month index calculation.
  - Update the template to render `displayTip.title` and `displayTip.advice`.
