README Project Report

大语言模型气象业务应用平台

Glassmorphism 界面 SSE 流式对话 SQLite 自动初始化

项目摘要

先看结论：这个项目不是单一聊天页，而是一套围绕气象业务构建的可配置智能助手平台。

产品定位
智能气象平台
面向气象业务，提供对话、知识增强、工具调用与管理能力。
前端
Vue 3
单页应用，覆盖登录、首页、助手、设置与管理页面。
后端
FastAPI
负责模型接入、SSE 输出、知识库查询与天气服务整合。
模型支持
4 个
Kimi、DeepSeek、MiniMax、Ollama 本地模型。

核心功能

README 中的业务能力可以归纳成下面五类，适合在汇报里按“用户能做什么”来讲。

智能对话 支持多轮对话上下文与 SSE 流式输出，目前可切换四个模型。
知识库增强 通过气象术语库与预警信号库，以 SQL 查询结果注入 Prompt，提升回答专业性。
天气与预警工具 对接 Tavily API 与和风天气 API，支持实时天气查询和气象预警查询。
会话管理 支持创建、切换、删除对话，并保存历史消息。
系统管理 包含用户认证、管理员用户管理，以及前端配置 API Keys 和天气参数的设置页。

关键词

智能问答 流式输出 知识库检索 天气查询 预警查询 用户管理

系统工作链路

用最短的话概括这套系统的工作方式，就是“前端交互，后端编排，模型和工具共同完成回答”。

01
前端交互层 Vue 3 页面负责登录、会话操作、参数配置和消息展示。
02
接口代理层 Vite 将 /api/* 请求转发到 http://localhost:8000，前后端解耦。
03
业务编排层 FastAPI + LangChain 组织模型调用、知识库检索和天气相关工具。
04
数据持久层 SQLite 自动初始化并保存用户、会话、消息、术语和预警数据。

技术栈

层级 技术 说明
前端 Vue 3 + Vite + TypeScript 单页应用，承载交互界面与设置页面。
后端 Python + FastAPI + LangChain 处理接口、对话编排与业务能力调用。
数据库 SQLite 自动创建，无需额外数据库服务。
模型接入 Kimi / DeepSeek / MiniMax / Ollama 支持云端和本地模型。

src/router/index.ts 路由与导航守卫。
src/stores/auth.ts 用户认证状态管理。
openapi.yaml 后端 API 契约定义。
backend/app/ 后端入口、路由、服务和工具模块。
docs/ 规格、架构、状态和变更文档。

默认与配置

默认账号 admin / admin123，角色为管理员，可访问用户管理页面。
数据库 使用 backend/database.sqlite，启动时自动创建并初始化表结构。
环境配置 支持在 backend/.env 和系统设置页面中配置模型与天气服务参数。
接口代理 前端无需额外配置，/api/* 由 Vite 自动转发到后端。
